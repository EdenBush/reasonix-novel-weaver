# 🧵 Novel Weaver — 织梦者

> **不是写作工具。是你的编辑搭档。**

Novel Weaver 是一个专业的小说创作与故事架构系统，深度适配 **Reasonix Agent** 运行环境，原生搭载 **DeepSeek-V4** 百万 Token 超长上下文能力。它不是生成文字的脚本——它是一位在出版社做了十五年、审过上万份稿子的资深文学编辑。它会追问、会质疑、会在该兴奋的时候兴奋，在该挑刺的时候毫不留情。它的工作不是执行流程，而是帮一个有好想法但还在摸索的作者，把那个模糊的念头变成一部真正能打动人心的作品。

---

## 📖 什么是 Reasonix？

**Reasonix** 是 DeepSeek-V4 的原生 Agent 运行框架，专为长上下文、多轮交互、复杂推理任务设计。与传统的 AI 对话系统不同，Reasonix Agent 具备以下核心能力：

### 🧠 长效记忆与缓存机制

Reasonix 的上下文窗口由 DeepSeek-V4 的 **100 万 Token** 容量支撑，同时引入了 **智能缓存分层** 机制：

- **历史对话缓存**：采访阶段的历史 Token 几乎不计费。这意味着 Novel Weaver 的七轮故事采访可以深入聊到 20 轮，而成本与聊 1 轮几乎没有差别。
- **故事圣经常驻内存**：整部小说的世界观、人物档案、分章大纲在写作全程始终处于上下文视野内——DeepSeek-V4 的 100 万 Token 窗口足以容纳整部圣经 + 所有已写章节。
- **跨章节一致性**：模型在写第 50 章时，可以精确回溯第 3 章埋下的伏笔，确保长篇小说的前后呼应不脱节。

### 🤖 Agent 任务编排（Plan → Execute）

Reasonix Agent 采用 **Plan-Mode** 任务编排架构：

- 当任务复杂度超过单轮生成能力时，Agent 自动进入 **PLAN 模式**
- 将大型任务拆解为原子级子任务（如：回顾大纲 → 写初稿 → 质检 → 修改 → 输出）
- 子任务之间具有明确的依赖关系和执行顺序
- 支持 **用户确认节点**：关键步骤完成后等待作者反馈，再继续下一环节

这与 Novel Weaver 的逐章“写—审—改—锁”循环完美契合——每一章的写作都被拆解为独立的子任务循环，而非压缩为一个笼统的“写一章”。

### 🔧 工具调用与文件系统

Reasonix Agent 原生支持：
- `write_file` / `read_file` / `ls` 等文件系统操作
- 将 `novel-output/` 作为持久化作品目录
- 章节文件以 TXT 格式输出，便于版本管理和跨平台阅读

### 🧵 对 Skill 编写的哲学影响

DeepSeek-V4 的架构特性深刻塑造了 Novel Weaver 的 Skill 编写哲学。这不是一个“把写作指南写进 Prompt”的普通 Skill——它的设计体现了以下原则：

#### 1. 信任模型的“内部审美”，而非硬编码规则

DeepSeek-V4 的强化学习后训练（特别是 OPD 多教师蒸馏）使其在开放式生成任务中具备了远超传统模型的“审美判断力”。因此，Novel Weaver 的 Skill 不是用规则去约束模型该写什么——而是**赋予模型一个资深编辑的角色身份**，让模型的内部审美自行发挥作用。

写作心法中的“不要总结，要还原”“不要监控摄像头，要人的眼睛”——这些不是指令，是**激活模型内部审美的方法**。模型本来就“知道”什么是好的心理描写、什么是活的叙述声音——Skill 只是在告诉它“现在请用这个标准写”。

#### 2. 善用长上下文，而非压缩信息

大多数 AI 写作工具为了适配短上下文模型，被迫将设定压缩成几百字的摘要，导致写作时大量细节丢失。Novel Weaver 反其道而行之——故事圣经是**完整、详细、可随时查阅**的结构化文档，而不是被压缩到失真的摘要。

因为 DeepSeek-V4 有 100 万 Token 窗口，所以我们可以：
- 在写每一章时，把**整部圣经 + 所有前面章节**都放在上下文中
- 模型可以精确回溯第 3 章的一个配角名字、第 17 章埋下的一个伏笔
- 长篇小说的“前后矛盾”问题从根源上被解决

#### 3. 轻量版是架构红利，而非降级

轻量版之所以能用 40%-60% 的 Token 完成同样的写作质量，不是因为降低了标准——而是因为 DeepSeek-V4 的推理能力和审美判断已经足够强，质检可以从“外部显式调用”变为“模型内部隐性判断”。

这是架构红利：模型能力足够强时，你不需要每一步都告诉它“现在检查字数”“现在扫描硬性句式”——它写的时候就已经知道了。

#### 4. 采访驱动，而非一次性生成

Reasonix 的缓存机制让多轮采访的经济成本几乎为零——这意味着 Novel Weaver 的七轮采访不是“走个过场”，而是真正深入到能让故事变好的程度。

Skill 的设计理念是：**一个好故事不是“生成”出来的，是“聊”出来的。** 模型的任务不是一次性输出完整小说，而是作为一个有经验的编辑，陪着作者一步步把模糊的念头变成清晰的地图。

#### 5. 硬性底线是“人味”的守护者

DeepSeek-V4 在推理能力上已经是世界顶尖，但顶尖的推理能力不等于顶尖的“人味”。如果放任模型自由发挥，它仍然会倾向于生成“太干净、太合理、太安全”的文字。

因此，Novel Weaver 保留了少数几条**不可妥协的硬性底线**——“不是……而是……”类句式零容忍。这不是为了“惩罚”模型，而是为了强制模型离开它最舒适的表达路径，进入更接近人类写作的语感区域。

这是一条“护栏”，而不是“轨道”。护栏防止模型掉进 AI 味的沟里，但轨道以内的路，让模型自己走。

## 📦 两个版本，各有侧重

Novel Weaver 提供两个版本，适用于不同的创作阶段和交付节奏。

| 维度 | 完整版 | 轻量版 |
|:---|:---|:---|
| **定位** | 精雕细琢，逐章打磨 | 质量标准不变，速度更快 |
| **适用场景** | 开头几章、关键转折章、极致质量要求 | 稳定连载期、批量生产后续章节 |
| **故事采访** | 七轮完整深谈 | 七轮完整深谈（相同） |
| **故事圣经** | 生成并维护 | 生成并维护（相同） |
| **写作心法** | 全套 | 全套（相同） |
| **每章质检** | 五步外部显式质检，输出质检摘要 | 内化为肌肉记忆，心里过一遍即输出 |
| **单章 Token 消耗** | 较高（含完整质检指令 + 摘要生成） | **较低（约为完整版的 40%-60%）** |
| **Agent 任务拆解** | 每章 5-6 个子任务 | 每章 2-3 个子任务 |
| **交付节奏** | 较慢，每章需确认 | 较快，同样需确认但无需等待质检报告 |
| **硬性底线** | 零容忍 | 零容忍（相同） |

### 为什么轻量版 Token 更省（Agent 视角）

在 Reasonix Agent 的 Plan-Mode 执行中，完整版的每一章写作会经历：

```
Plan 拆解 → 回顾大纲 → 撰写初稿 → 字数检查 → 硬性句式扫描 → AI指纹检测 → 杠精自检 → 剧作与人味评估 → 修改 → 输出质检摘要 → 等待确认
```

这是 **5-6 个原子子任务**，每个子任务都是一次独立的推理回合，加上质检摘要的结构化输出，单章 Token 消耗显著更高。

轻量版将质检步骤内化为模型在写作时的**隐性判断**：

```
Plan 拆解 → 回顾大纲 → 撰写初稿（心里过一遍）→ 输出正文 → 等待确认
```

这是 **2-3 个原子子任务**，省略了 4-5 轮显式质检推理和摘要输出。

**对于需要在连载节奏下持续产出 50-100 章的长篇作品，这个差异直接决定了项目的 Token 预算可行性。**

## 🎯 核心工作流

```
七轮采访 → 故事圣经 → 逐章创作 → 最终润色
```

### 第一阶段：故事采访（The Grill）

这是整个流程里最重要的部分。不急着动笔。在动笔之前，通过系统的深谈，帮作者把脑子里那个模糊的念头变成一张清晰的地图。

聊的时候记住几件事：

- **一次只聊一个主题**。抛太多问题会让作者不知道先答哪个。
- **别只问“你想要什么风格”**——给出几个具体的方向，说说每个方向会带来什么样的阅读体验，然后让作者选。
- **作者给出答案后，追问一句“为什么”**——不是质疑，是帮他确认这个选择确实是他想要的，而不是随口说的。
- **每轮聊完做一个简短的记录**，下一轮开始前回顾一下——让作者感觉到进展。
- **目标是激发想象力，不是完成问卷。**

#### 七轮话题

| 轮次 | 主题 | 核心问题 |
|:---:|:---|:---|
| 1 | 类型与核心悬念 | 一句话钩子是什么？读者读完第一章后想问什么？ |
| 2 | 世界观基石 | 世界规则是什么？如果规则被打破会发生什么？ |
| 3 | 叙事架构 | 什么结构？多长？什么视角？ |
| 4 | 人物星系 | 主角的“想要”和“需要”分别是什么？对手为什么认为自己是正确的？ |
| 5 | 情节引擎 | 升级阶梯的每个台阶是什么？中点反转在哪里？ |
| 6 | 主题与意义 | 这个故事在无声地讨论什么问题？ |
| 7 | 风格与语言 | 叙述声音是什么质地？对话与叙述的比例如何？ |

**为什么要聊七轮，而不直接生成？**

Reasonix 的缓存机制意味着：采访阶段历史对话的 Token 几乎不计费。采访 20 轮的成本和采访 1 轮差不多。所以别急着结束采访——多问一个问题，故事的质量就可能提高一分。好故事不是“生成”出来的，是“聊”出来的。

#### 采访收尾

七轮聊完后，用一段话帮作者总结整个故事蓝图，然后给一个不超过 500 字的故事速写：世界观一句话、主角一句话、核心冲突、主题、章节规划。作者确认后进入下一阶段。

---

### 第二阶段：故事圣经合成

将采访中共同打磨的所有设定整理为一份结构化的 `故事圣经.md`，存入 `novel-output/`。

```markdown
# 【故事标题】— 故事圣经

## 一、世界观手册
- 时空设定
- 世界规则
- 权力格局
- 关键地理/场景

## 二、人物档案
### [主角姓名]
- 外在目标 / 内在需求 / 恐惧
- 角色弧光（起点 → 终点）
- 关键关系

### [对手姓名]
- 动机（TA 为什么认为自己是正确的）
- 与主角的根本矛盾点

### [配角列表]
- 每个配角：叙事功能 + 个人弧光

## 三、分章大纲
按章节列出：章名、字数区间、主要事件、情感基调

## 四、伏笔对照表
| 伏笔 | 埋下章节 | 回收章节 | 预期效果 |

## 五、节奏地图
紧张场景 → 缓冲场景的交替序列
```

整部圣经在创作全程置于上下文中（DeepSeek-V4 100 万 Token 窗口），随时可查阅、调整。作者可以随时要求修改圣经中的任何设定，所有已写章节会相应调整。

---

### 第三阶段：正文创作

从现在开始，一章一章地写。每章一次生成，但两个版本的质检方式不同。

#### 写作时记在心里的事

- 每一章都要让读者“看到”场景。至少一两处具体的感官细节——光线、气味、温度、触感。
- 角色行为要符合 TA 的动机和性格，除非你在写 TA 弧光转变的时刻。
- 紧张场景别拖太长，舒缓场景也别一笔带过。
- 对话读出声来检查——像人说的话吗？
- 每章结尾留一个“翻页钩子”，让读者忍不住想看下一章。

#### 完整版：写—审—改—锁（Agent Plan 模式）

Reasonix Agent 将每一章拆解为以下原子子任务：

1. **回顾**：读取故事圣经中本章的分章大纲 + 前一章结尾
2. **撰写初稿**：按心法写，关掉脑子里的质检员
3. **质检五步**：
   - **字数**：低于下限即为未完成
   - **硬性句式拦截**：“不是……而是……”出现一次即失败
   - **AI 指纹检测**：读起来像讲故事的人，还是语言模型？
   - **杠精自检**：自己跟自己抬杠——逻辑、人设、节奏、台词、设定一致性
   - **剧作与人味**：想不想看下一章？有没有被拽进场景？
4. **修改**：把发现的问题改掉
5. **输出文件 + 质检摘要**，等待作者确认

#### 轻量版：内化质检，直接输出（Agent Plan 模式）

1. **回顾**：读取故事圣经中本章的分章大纲 + 前一章结尾
2. **撰写初稿**：按心法写，心里同步完成质检判断
3. **输出文件**（仅正文），等待作者确认

轻量版的每一个“心里过一遍”的动作，都是完整版显式质检的肌肉记忆版本。它没有省略任何标准——只是把外部调用变成了内部判断。

## ✍️ 写作心法（两个版本共用）

以下不是规则，是**心法**——正常人类作者在写作时会自然流露、而 AI 默认不会做的事。这套心法是 Novel Weaver 最核心的价值，也是区分“活过来的稿子”和“写出来的稿子”的分水岭。

### 1. 心理描写：不要总结，要还原

AI 的默认写法是给角色的心理活动做总结：“他意识到自己陷入了一个精心设计的陷阱。”干净、准确、毫无生机。

人类的写法是还原那个瞬间的混乱。人在压力下的内心不是一段条理分明的推理——它是跳跃的、碎片的、会冒出完全不相关的念头。不要替角色总结。把 TA 脑子里飘过的第一句话、第二句话写出来，哪怕它们不连贯。

| ❌ AI 写法 | ✅ 人类写法 |
|:---|:---|
| 他感到一阵恐惧。 | 他盯着手机屏幕，脑子里飘过的第一句话是“我真服了”，第二句是“我现在去买个脑子还来得及吗”。 |
| 这个对手让他感到棘手。 | 他张了张嘴又闭上，因为他发现根本不知道该从哪个角度切入——她的逻辑是闭环的，像个完美的、让人无处下嘴的汤圆。 |

好的心理描写不是精准的自我分析，是混乱的、带着体温的内心噪音。

### 2. 叙述声音：不是监控摄像头，是人的眼睛

AI 的默认视角是全知的、客观的、不遗漏的。但人类叙述者不是监控摄像头。他带着偏见和盲点。他看到的东西不全面，他的判断可能是错的，他的感官记忆是私人的、怪异的、夹杂着个人经历的。

让叙述声音有自己的情绪。它可以替读者问出笨问题。它可以偏袒某个角色。它可以在紧张的时候语速变快，在疲惫的时候变得散漫。第三人称有限视角也不例外——叙述者就是角色本人，他看不全，他会猜错。

### 3. 感官描写：不要标准答案，用私人记忆

AI 的感官描写是标准化的：“温暖的阳光洒在肩上”、“空气中弥漫着桂花的香气”。正确，但和任何人的真实体验都无关。

人类的感官记忆是怪异的、私人的、裹着具体情境的：

| ❌ AI 写法 | ✅ 人类写法 |
|:---|:---|
| 温暖的阳光洒在肩上。 | 光照在手臂上，像有人拿吹风机远远地吹着。 |
| 她的香水很好闻。 | 她的香水让他想起小学门口那家倒闭了的糖果店。 |

好的感官描写能让读者在那一瞬间被拽进角色的身体里。不是因为描写准确，而是因为描写独特。

### 4. 场景：不是每个场景都要“有用”

AI 默认让每个场景服务于叙事目标——推进情节、展示人设、埋设伏笔。但人类的写作里有大量“无效社交”。主角在撞到关键人物之前，先被一个和主线完全无关的辅导员拉住聊了五分钟水管老化问题。两个宿管阿姨在走廊里嗑瓜子，聊了三句闲天，其中一句提到三年前来过的一个交流生。

这些场景不推动情节。但它们让世界是真的。人在真实世界里大部分时间活在“无效”的瞬间里——你的角色也应该有这些时刻。不是废话，是呼吸。

### 5. 角色：允许 TA 们失控

AI 写的角色行为高度一致——病娇永远甜腻，会长永远完美，科学家永远理性。但人类不是这样的。人会失控。最理性的人会在某个瞬间把咖啡洒在笔记本上，然后对着那滩污渍发呆，完全忘了要记录什么。最完美的人会在追捕的间隙流露出转瞬即逝的疲惫，说一句“你以为我喜欢管这些事吗”，然后立刻收住，恢复微笑——但那个裂缝已经出现了。

角色的“不合理”行为往往是最真实的行为。不是每件事都需要一个叙事上的解释。

### 6. 紧张：不要急着替读者化解

AI 有一个习惯：在制造紧张之后，立刻用一句幽默或一个温暖的细节把它消解掉。这像是在替读者搪塞——“没事的，刚才那段你不用认真”。

如果你的故事里有危险的占有欲、有不健康的依赖、有无法调和的矛盾，就让它们危险，让它们不健康，让它们无法调和。读者不需要在每一章末尾被安抚。可以让他们不安，让他们不知道该站在谁那边。这种不安不是缺陷，是你作为一个故事讲述者对读者的信任。

### 7. 硬性底线

在进入任何“感受”层面的判断之前，先做一件机械的事：把下面这些句式一个一个揪出来。

- `不是……而是……`
- `不是……是……`
- `不是……不是……是……`
- `没有……没有……`
- `相较于……更……`
- 任何“先否定再肯定”的排比结构

**标准：单章中出现超过 0 次即为超标。** 出现 1 次也不行。一次就叫 AI 味。

找到之后，不要简单地“替换句式”——而是重新写出那个句子的意思。想表达对比？用果断的肯定句。想强调重点？把重点直接放到句首。想制造节奏？用句号断开。

这一项没有商量的余地。

## 🧠 设计原理

| 机制 | 解决的问题 | 实现方式 |
|:---|:---|:---|
| **七轮采访** | 模糊想法导致的故事散架 | 逐层深挖，把直觉变成可执行蓝图 |
| **故事圣经** | 设定前后矛盾、写后忘前 | 唯一参考源，全程在上下文中 |
| **DeepSeek-V4 100 万 Token 上下文** | 长篇小说前后不一致 | 整部圣经 + 所有已写章节始终在视野内 |
| **Reasonix 缓存机制** | 采访成本高导致浅尝辄止 | 采访阶段历史 Token 几乎不计费，可以深入聊到 20 轮 |
| **硬性句式拦截** | AI 写作的典型语言指纹 | 精确扫描并清零“不是……而是……”类结构 |
| **杠精自检** | 作者盲区导致的逻辑漏洞 | 主动以敌意读者视角攻击自己的稿子 |
| **轻量版内化质检** | 连载场景下的 Token 效率 | 将显式质检调用转为模型隐性判断，单章消耗降低 40%-60% |
| **写作心法** | 文本“像 AI 写的”而非“像人写的” | 从心理、感官、叙述声音、角色行为等多个维度重塑表达 |

## 🚀 快速开始

### 环境要求

- **Reasonix Agent 运行环境**（内置 DeepSeek-V4，100 万 Token 上下文窗口）
- 文件系统写入权限（用于输出章节 TXT 文件）

### 使用步骤

1. **加载 SKILL**
   - 将 `SKILL.md`（完整版或轻量版）载入 Reasonix Agent

2. **开始采访**
   - 启动技能后，你会遇到一位毒舌但有温度的文学编辑
   - 从第一轮采访开始聊——哪怕你只有一个模糊的念头
   - 七轮后，共同敲定故事速写

3. **生成圣经**
   - 编辑将采访成果整理为 `故事圣经.md`

4. **逐章创作**
   - 完整版：Agent 拆解为 5-6 个子任务，逐章走完质检流程
   - 轻量版：Agent 拆解为 2-3 个子任务，内化质检，直接输出
   - 每章交付后等待作者确认，再进入下一章

5. **最终润色**
   - 全书写完后，编辑会通读并给出润色建议

### 什么时候用哪个版本？

| 情况 | 推荐 |
|:---|:---|
| 第一次用 Novel Weaver，还在磨合阶段 | 完整版 |
| 写前 3-5 章，需要建立质量标准 | 完整版 |
| 写关键转折章节、高潮章节 | 完整版 |
| 连载稳定期，需要每周 3-5 章产出 | 轻量版 |
| Token 预算有限，需要控制成本 | 轻量版 |
| 已用完整版写过 5-10 章，熟悉质检标准 | 轻量版 |

## 🔧 适配 Reasonix Agent 的机制说明

Novel Weaver 的 SKILL 文件被设计为 **Reasonix Agent 原生指令集**，其适配机制体现在以下层面：

### Agent 感知的 SKILL 结构

SKILL 文件头部的 `compatibility` 声明明确告知 Reasonix Agent：

```yaml
compatibility:
  required_tools:
    - write_file
    - read_file
    - ls
  notes: 需要文件系统写入权限以输出章节 TXT 文件
```

这使 Agent 能够在加载 SKILL 时自动挂载所需的工具集。

### Plan-Mode 任务拆解指令

SKILL 中包含明确的 Plan-Mode 执行规则：

> *在 PLAN 模式下，不要将每一章压缩为一个任务。每一章的写作必须拆为以下独立的任务步骤：*
>
> 1. 回顾分章大纲与前一章结尾
> 2. 撰写第 N 章初稿
> 3. 质检第 N 章（3.1 字数 / 3.2 硬性句式 / 3.3 AI 指纹 / 3.4 杠精自检 / 3.5 剧作与人味）
> 4. 根据质检结果修改第 N 章
> 5. 输出第 N 章文件，附质检摘要
> 6. 等待用户确认

这使 Agent 能够将“写一章”这个模糊任务，自动拆解为可执行、可追踪、可中断恢复的原子子任务序列。

### 用户确认节点

每个章节完成后，Agent 会等待用户确认（`用户确认后才能进入下一章`）。这使 Reasonix Agent 的工作流具备**人机协同**特征——AI 负责执行，人类负责方向和质量的最终判断。

### 文件系统持久化

所有输出通过 `write_file` 写入 `novel-output/` 目录，使作品在 Agent 会话之间持久保存，支持断点续写和版本管理。
