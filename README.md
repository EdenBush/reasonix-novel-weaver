# Reasonix Novel Weaver — AI 长篇小说创作技能体系

一个为 **DeepSeek V4** 架构深度优化、由 **Reasonix Agent** 驱动的模块化长篇小说创作引擎。它解决的不仅是“让AI写小说”，而是**如何通过系统化的工程手段，消除AI写作的“机器感”，将长篇故事创作从灵感的随机碰撞，转化为质量可控、风格稳定的工业化流程**。

## 为什么需要这样一套体系？

当前的大语言模型能写出文笔流畅的段落，但难以驾驭数十万字的长篇叙事，且输出常带有一种“AI味”——结构过于工整、人物功能化、语言平滑缺乏质感、情感表达安全且可预测。

**这不是模型能力的问题，而是调用方式的问题。** 通用的、指令式的Prompt会激活模型内部“执行任务”的专家，输出安全、正确、平滑的文本；而我们需要的是激活“文学创作”的专家，产出有毛边、有温度、有意外之喜的叙述。

这套体系的核心贡献在于：**它是一套专门为激活DeepSeek V4文学创作潜能而设计的“操作系统”。** 通过**人格化角色设定**、**分层式写作心法**、**硬性/软性质检分离**以及**模块化流程协作**，这套体系将DeepSeek V4从“文字生成器”转变为一个具备“编辑直觉”和“创作韧性”的专业故事创作伙伴。

## 技术根基：与DeepSeek V4架构深度适配

这套体系是专为 **DeepSeek V4** 的架构特性设计的。它对以下技术特性进行了针对性利用：

### 1. MoE路由与“人格化指令”的科学依据

DeepSeek V4基于Mixture-of-Experts (MoE) 架构，拥有海量参数但每个token仅激活部分专家。系统包含384个路由专家，每个token激活其中6个，另有1个共享专家。**路由网络决定了哪些专家处理当前输入。**

当输入是一份产品说明书式的指令（“步骤一：确定故事类型；步骤二：构建人物小传……”）时，模型会自动将计算资源分配给那些擅长“指令遵循”和“结构化输出”的专家。这会导致输出虽然准确、流畅，但缺乏文学性。

**本体系的所有技能，都将自身设定为一个具体的人物角色：**
- **Weaver核心写作引擎**被设定为“在出版社做了十五年的资深文学编辑”。
- **去AI味修复师**被设定为“泡在书堆里活了二十年的老书虫”。
- **节奏诊断师**被设定为“专门给长篇小说号脉的医生”。

**这不是修辞，而是对MoE路由的精确操控。** 当模型读取到这样的人格化设定时，输入上下文中的词汇域、语气和身份信息，会引导路由网络将计算任务分配给那些在文学语料、叙事分析、人物洞察等任务上训练出来的专家。

### 2. OPD蒸馏与“对比式引导”的效用

DeepSeek V4的后训练采用了On-Policy Distillation (OPD)，将数学、编程、指令遵循等多个领域的专家模型知识蒸馏到一个统一模型中。这意味着**模型内部天然存在着多种“专家分布”的混合**。

本体系大量使用**对比式引导**而非规则清单。例如，写作心法中会写：“AI通常会写‘他感到一阵恐惧’；但人类会写‘他把手机翻过来扣在桌上。屏幕朝下。然后去厨房倒了一杯水。他忘了关水龙头。’”

这种对比在效果上相当于**在调用时临时调整OPD中不同专家分布的权重**，抑制“通用语言模型”分布的权重，增强“人类文学表达”分布的权重，从而让输出向更具“人味”的方向偏移。

### 3. GRPO的副作用与“松绑”机制

DeepSeek V4在强化学习阶段使用了GRPO算法，通过奖励模型引导模型生成符合标准的输出。这带来了一个副作用：**模型倾向于选择统计上最安全、最合理、最不具冒犯性的表达**——即“写对”，而不是“写活”。

本体系的核心“写作心法”和标志性的“关掉质检脑”机制，本质上是在对GRPO形成的保守倾向进行对抗。在写作阶段，系统主动提示模型：“允许角色跑题、失控、说废话”、“不要急着化解所有紧张”、“现在你只是一个讲故事的人”。这些“松绑”指令降低了安全专家和指令遵循专家的路由权重，为文学专家释放了更大的自由度。

### 4. CSA/HCA长上下文与全流程追踪

DeepSeek V4通过压缩稀疏注意力(CSA)和重度压缩注意力(HCA)技术支持100万token上下文窗口，且推理成本极低。这使得本体系得以在设计上要求：**整个“故事圣经”、所有已写章节、所有质检反馈，都全程保留在上下文中。**

配合Reasonix Agent的前缀缓存机制（append-only模式，历史对话几乎不计费），深度采访、逐章质检和大规模上下文管理不再有性能或经济上的障碍。

## 核心设计哲学：我们如何对抗“AI味”

在长期实践中，我们沉淀出以下设计哲学，作为所有技能开发的最高准则：

### 哲学一：人格化身份，而非操作手册
严禁任何“步骤1、步骤2”的指令式编写。所有技能都必须赋予一个具体、有经验感的人格，通过角色扮演让模型进入正确的“专家模式”。

### 哲学二：松绑，而非约束
不增加“确保包含X个元素”的规则，而是下达“允许做Y”的许可。通过减少约束来释放创造力，对抗模型趋于保守的本能。

### 哲学三：对比式引导，而非规则清单
不告诉模型“怎么写更好”，而是展示“AI通常会这样写，而人类通常会那样写”。通过方向感替代精确控制，让模型在统计分布上发生偏移，而不是僵化地执行一条硬性规则。

### 哲学四：创作与质检状态严格分离
写作时必须关掉“质检脑”，让模型完全沉浸在创作状态中。所有硬性质检（字数、句式、圣经一致性）必须作为独立的模块，在创作完成后由另一个“人格”执行。这避免了模型在创作时因预知审查而进行自我审查，导致文本重回“安全平滑”状态。

### 哲学五：模块化协作，各司其职
不试图将所有功能集成在一个巨大的脚本中。每个子技能负责一个单一任务，拥有独立的人格和工作流程。这确保了每个模块都足够轻量，Agent在执行时不易因指令过长而出现注意力衰减。

## 核心引擎：Weaver — 全流程长篇小说写作技能

`reasonix-novel-weaver` 是整个体系的**灵魂和绝对核心**。它不止是一个“写作工具”，它是一个**浓缩了完整长篇创作流程的独立技能**。它可独立使用，无需依赖其他子技能，即可完成从故事构思到正文创作的全过程。

Weaver的核心目标是将模糊的创作念头，通过系统化的流程，转化为高质量的、具有人类作家风格的长篇小说章节。

### Weaver的角色设定：资深文学编辑
Weaver被设定为 **“一个在出版社做了十五年、审过上万份稿子的资深文学编辑”** 。这个身份贯穿始终，它不是一个生硬的“执行引擎”，而是用户的创作伙伴。它会追问、会质疑、在该兴奋的时候兴奋，在该挑刺的时候毫不留情。

### Weaver的独立四阶段流程

即使没有其他任何辅助技能，Weaver自身也能完成一部小说从构思到完结的全部工作。其内部流程如下：

#### 第一阶段：故事采访 (The Grill)
Weaver不会让用户直接动笔，而是扮演专业编辑的角色，通过七轮系统性的深度提问，帮助用户将模糊的想法打磨成完整的故事蓝图。这是整个流程的核心。
- **第1轮 类型与核心悬念**：明确故事心脏，找到那个让读者不得不点进来的“钩子”。
- **第2轮 世界观基石**：构建冰山水面之下的十分之九，确立世界运行的底层规则和权力格局。
- **第3轮 叙事架构**：选定最适合故事特质的叙事结构（三幕式、英雄之旅、多线叙事等），确定篇幅和视角。
- **第4轮 人物星系**：挖掘主角与对手的内外需求与恐惧，构建功能分明又各具弧光的配角阵容。
- **第5轮 情节引擎**：搭建主冲突的层层升级阶梯，设计“中点反转”，编制伏笔网络和节奏地图。
- **第6轮 主题与意义**：无声地探讨故事的核心命题，通过角色的弧光来体现主题的展现。
- **第7轮 风格与语言**：决定叙述声音（简约、繁复、诗化、冷峻、口语），确定对话与叙述的比例。

#### 第二阶段：故事圣经合成
采访完成后，Weaver将所有共识整合成一份**不可动摇的故事圣经**。它包含世界观手册、完整人物档案、40章以上的分章大纲、伏笔对照表和节奏地图。这份圣经是后续所有写作的“法律”。

#### 第三阶段：正文创作
Weaver提供了两种写作模式：
1.  **完整版**：一章一走“写—审—改—锁”的循环，包含独立的质检环节（字数、硬性句式、AI指纹扫描、杠精自检、剧作与人味评估）。这是Weaver自带的质量控制系统。
2.  **Lite版**：将质检内化到写作过程中，追求速度和低Token消耗。通过“写作心法”和“关掉质检脑”机制，让模型在放松状态下写出更自然的文本。Lite版是Weaver效率的极致体现。

在写作阶段，Weaver执行了一套独特的“写作心法”来对抗AI味：
- **心理描写**：不用“总结”，用“还原”。不写“他感到恐惧”，而写“他把手机翻过来扣在桌上，去了厨房，忘了关水龙头。”
- **叙述声音**：不用“监控摄像头”，用“人的眼睛”。叙述者带着偏见、盲点和私人记忆。
- **感官描写**：不用“标准答案”，用“私人记忆”。不说“温暖的阳光”，说“光照在手臂上，像有人拿吹风机远远地吹着。”
- **场景**：不是每个场景都要“有用”，允许“无效社交”来让世界更真实。
- **角色**：允许他们失控、跑题，做出不符合“人设”但更真实的反应。
- **紧张**：不要急着替读者化解，让他们在不安里多待一会儿。

#### 第四阶段：最终润色
全书完成后，进行一致性通读、节奏评估、开头重评估以及后记/番外的创作。

### Weaver的独立性与重要性
Weaver是整个体系**唯一可以独立完成全流程的核心技能**。所有其他子技能，本质上都是对Weaver流程中某个环节的**增强、修复或诊断**。用户完全可以只使用Weaver，就完成一部高质量小说的创作。

## 辅助工具集：为 Weaver 提供专业增强

Weaver本身已足够强大，但我们的工具集为它提供了更专业的“专科医生”服务。它们围绕Weaver搭建，在需要时进行会诊。

### 写作辅助与质量保障
- **Weaver Lite**：Weaver的高效分身，专注于快速产出章节。它将Weaver的质检环节内化，关闭“质检脑”，追求单次生成的极致流畅和“人味”。
- **去AI味修复师 (DeAI)**：专门清洗顽固AI痕迹的老书虫。当Weaver产出的章节经过人工审阅，仍被发现有“机器气息”时，它能分六步进行深度清洗：捉虫（硬性句式）、拆台（心理降级）、换眼（感官私人化）、破功（增加毛边）、放手（保留紧张）、点数（确认字数）。
- **补字匠 (Pad)**：当章节字数不达标时，它负责找到质感单薄之处，通过展开瞬间、添加细节来补足字数，不灌水。
- **硬质检员 (Hard Check)**：一个只负责数字数、扫句式、对圣经的机械检查员。它不关心文学性，只输出“通过”或“退回”，确保每章都达到交付底线。

### 质量增强与诊断
- **故事深度增强师 (Write Master)**：人称“书医”。专门解决章节“没毛病但差一口气”的问题。它从冲突的灰度、对话的潜台词、感官的私人性、情绪的延迟、细节的无用之用、紧张的留白等六个维度，对章节进行深度修改，把“能看”变成“好看”。
- **对话增强师 (Dialogue Master)**：一位老话剧导演。专门诊断和修复对话中的问题，为台词植入潜台词，强化角色性格标识，调整对话节奏，让角色“真的在聊天”。
- **节奏诊断师 (Rhythm Doctor)**：长篇小说的心电监护仪。它逐章分析节奏类型，画出情绪曲线，标记节奏单调、紧张疲劳或缓冲空洞的风险，只诊断不修改，为作者提供决策参考。
- **读者视角模拟器 (Reader Simulator)**：能够模拟普通读者、挑剔读者和作者同行三种身份，分别给出阅读感受，帮助作者获取多元视角，发现自己难以察觉的盲点。

### 工程化维护
- **圣经维护员 (Bible Updater)**：长篇项目的档案记录员。在每卷结束后，它会更新分章大纲为“实际态”，标记伏笔回收状态，追踪角色弧光进展，并列出所有需要关注的规划调整，确保圣经永远是一份“活文档”。
- **主编调度中心 (Orchestrator)**：整个体系的“管家”。它了解所有工具的能力，并在合适的时机主动向用户建议“现在该叫谁上场了”，帮助用户管理整个创作周期。

## 与 Reasonix Agent 的协作：让流程自动流转

这套体系是专为 **Reasonix Agent** 的append-only缓存架构设计的。

1.  **启动**：用户通过 `reasonix-novel` 唤醒主编调度中心，或直接启动 `weaver` 核心引擎。
2.  **规划**：Agent的PLAN模块会根据技能文件中的流程指令，自动生成待办列表（例如：采访用户 -> 合成圣经 -> 撰写第1章 -> 质检第1章 -> 修改 -> 输出 -> 等待确认）。
3.  **执行**：Agent顺序执行待办列表中的任务。当需要调用辅助工具时，它会自动打开 `tools/` 文件夹下的对应技能文件，以该工具的人格身份开始工作，完成后再切回主身份。
4.  **监控与维护**：在连续创作过程中，Agent会根据主编调度的逻辑，主动在合适的时机（如写完3-5章，或完成一卷）向用户建议进行节奏诊断、视角模拟或圣经维护。

## 用户最佳实践

1.  **从Weaver开始**：无论是新手还是老手，都可以直接使用 `Weaver` 开始工作。这是最完整、最能体现体系价值的入口。
2.  **善用Lite模式**：在故事走上正轨后，切换到 `Weaver Lite` 进行快速章节推进，可以有效降低Token消耗并提升产出速度。
3.  **手动触发专科医生**：当人工审阅发现特定问题时，直接呼叫对应的辅助工具。例如，觉得对话生硬，就叫 `Dialogue Master`；觉得章节没味道，就叫 `Write Master`。
4.  **信任硬质检**：不要跳过 `Hard Check`。它是防止Agent注意力衰减、保证章节质量底线的最后一道防线。
5.  **定期维护圣经**：养成每写完一卷就运行 `Bible Updater` 的习惯，避免长篇写到后期圣经与正文严重脱节。

## 价值总结

`Reasonix Novel Weaver` 体系不是一份简单的“写作Prompt”，而是一套**长篇小说创作的工业化解决方案**。它通过深入利用DeepSeek V4的架构特性，结合一套独特的设计哲学，将AI写作从“随机生成”提升为**过程可控、质量稳定、风格更具人性的专业创作活动**。

其核心贡献在于：**它证明并实践了——通过精心设计的“人格化系统调用”，我们可以精准地操控大规模语言模型内部的专家路由，从而在宏观上改变模型的输出风格，实现从“机器写作”到“人类写作感”的跨越。**
